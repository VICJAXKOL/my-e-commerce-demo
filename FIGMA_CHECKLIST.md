# MyShop Figma Checklist

## How To Use This

This checklist is for turning the redesign plan into actual Figma work.

Recommended flow:

1. Complete the foundations first
2. Build reusable components second
3. Design high-priority screens next
4. Review mobile and desktop versions before implementation

Use this as a working checklist, not just a reference document.

---

## Phase 1 — Foundations

### File Setup

- [ ] Create a Figma file named `MyShop Redesign`
- [ ] Create pages for `Foundations`, `Components`, `Marketing`, `Shopping`, `Auth`, and `Notes`
- [ ] Add a simple naming convention for frames and components

### Color System

- [ ] Define primary brand color
- [ ] Define primary hover and active colors
- [ ] Define neutral background colors
- [ ] Define surface colors for cards and panels
- [ ] Define border colors
- [ ] Define primary and secondary text colors
- [ ] Define success, warning, and error colors
- [ ] Check color contrast for accessibility

### Typography

- [ ] Choose heading font behavior and sizes
- [ ] Define display text style
- [ ] Define page title style
- [ ] Define section heading style
- [ ] Define card title style
- [ ] Define body text style
- [ ] Define secondary text style
- [ ] Define caption and helper text style
- [ ] Define button text style

### Spacing

- [ ] Define spacing scale: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64
- [ ] Decide section spacing for desktop
- [ ] Decide section spacing for mobile
- [ ] Define card internal padding sizes
- [ ] Define form field spacing

### Radius and Elevation

- [ ] Define small radius
- [ ] Define medium radius
- [ ] Define large radius
- [ ] Define extra-large radius
- [ ] Define subtle card shadow
- [ ] Define hover shadow
- [ ] Define sticky panel / elevated panel shadow

### Grid and Layout

- [ ] Define desktop content width
- [ ] Define tablet behavior
- [ ] Define mobile margins
- [ ] Define grid rules for cards and sections
- [ ] Decide how sticky elements behave on large screens

## Phase 3 — Priority Screens

### 1. Home Page

#### Required Work

- [ ] Design desktop home page
- [ ] Design mobile home page
- [ ] Create stronger hero layout
- [ ] Reduce text-heavy sections
- [ ] Add featured shopping entry points
- [ ] Define clearer CTA hierarchy
- [ ] Improve section rhythm and spacing
- [ ] Add stronger visual merchandising blocks

#### Review Questions

- [ ] Is it obvious what the store sells in 5 seconds?
- [ ] Is the primary CTA clear?
- [ ] Does the page feel like a store, not a documentation page?
- [ ] Is the visual hierarchy consistent?

### 2. Product Listing Page

#### Required Work

- [ ] Design desktop product listing
- [ ] Design mobile product listing
- [ ] Redesign search area
- [ ] Redesign filters and sort controls
- [ ] Improve product grid spacing
- [ ] Improve empty state
- [ ] Improve product card scannability

#### Review Questions

- [ ] Can users scan products quickly?
- [ ] Are filters easy to understand?
- [ ] Is the card CTA hierarchy clear?
- [ ] Does the page feel premium rather than cramped?

### 3. Product Detail Page

#### Required Work

- [ ] Design desktop product detail page
- [ ] Design mobile product detail page
- [ ] Improve gallery/media layout
- [ ] Redesign purchase information panel

#### Review Questions

- [ ] Is the add-to-cart path immediately obvious?
- [ ] Can users understand the product quickly?
- [ ] Do trust signals feel credible and well placed?
- [ ] Is the purchase panel visually strong enough?

## Responsive Review Checklist

Use this for every priority screen.

- [ ] Desktop version designed
- [ ] Tablet behavior considered
- [ ] Mobile version designed
- [ ] Buttons remain easy to tap on mobile
- [ ] Text remains readable on small screens
- [ ] Cards do not feel overcrowded
- [ ] Sticky or side panels degrade gracefully on mobile

---

## Handoff Checklist For Code

Before implementation begins:

- [ ] Colors are final
- [ ] Typography styles are final
- [ ] Spacing scale is final
- [ ] Buttons and inputs are final
- [ ] Desktop and mobile designs are both available
- [ ] Interaction notes are written
- [ ] Components that repeat across screens are identified

- [ ] Buttons
- [ ] Inputs
